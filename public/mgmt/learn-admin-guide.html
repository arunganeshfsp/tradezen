Learn Admin — Editor Guide

A complete guide to creating and publishing lesson content on the TradeZen learn platform. No technical knowledge required.

This page walks you through everything you need to build a lesson — from filling in chapter details to adding cards, quiz questions, and publishing to learners. Use the table of contents on the left to jump to any topic.

📍 Where to find the admin page Open your browser and go to: https://tradeze.in/mgmt/learn-admin.html
1
Logging In
The admin page requires a token to protect it from unauthorised access.

When you first open the page, a prompt appears asking for your Admin Token. This is a password set by your server administrator.

Browser prompt
Enter ADMIN_TOKEN:
••••••••••••
OK Cancel
  • 1
    Type your Admin Token exactly as given to you. It is case-sensitive.
  • 2
    Click OK. The token is saved for this browser session — you won't be asked again until you close the tab.
  • 3
    If you need to change the token later, click the 🔑 Token button in the top-right corner.
⚠️ Wrong token? The tree will fail to load and you'll see an error toast at the bottom-right. Click 🔑 Token, re-enter the correct token, then refresh the page.

Adding Cards

There are three card types. A typical lesson uses one Cover, several Content cards, and one Result card.

🎬
Cover Card
The first screen learners see. Set the title, description, and emoji.
📄
Content Card
The main teaching content. Add as many blocks as you need inside each card.
🏆
Result Card
Auto-generated completion screen. Add it at the end — no fields to fill.
  • 1
    Click + Card in the gap where you want to add the card.
  • 2
    A new card appears at that position. The default type is Content.
  • 3
    Click the card header to expand it, then fill in the fields.

Adding Blocks Inside a Content Card

Each Content Card holds a list of blocks. Blocks are the individual pieces of content — a paragraph of text, a tip, a table, a comparison. You can mix and match any combination.

📄 CONTENT CARD — expanded
¶ Text
Think of it like a bank locker…
இதை ஒரு வங்கி லாக்கர் போல்…
💡 Tip
Popular broker apps in India…
இந்தியாவில் பிரபலமான broker ஆப்கள்…
¶ Text
+ Add Block
  • 1
    Inside an expanded Content Card, find the block type dropdown at the bottom (e.g. ¶ Text).
  • 2
    Choose the block type from the dropdown — see the Block Type Reference below for guidance on which type to use.
  • 3
    Click + Add Block. The block appears at the bottom of the card.
  • 4
    Fill in the English and Tamil fields side by side.
  • 5
    To remove a block, click the ✕ button on the right of the block header.
💡 How many blocks per card? 2–4 blocks per card keeps the lesson readable on mobile. If you have more content, add a second Content Card instead of cramming everything into one.

Inserting Quiz Questions

Quiz questions appear between cards — like checkpoints in the lesson flow. A good lesson places a quiz after every 1–2 content cards.

  • 1
    Find the gap between two cards where you want the quiz. Click the gold + Quiz button.
  • 2
    Choose the question type: Multiple Choice (2–6 options) or True / False (auto-fills two options).
  • 3
    Type the question in both English and Tamil.
  • 4
    Fill in each option in English and Tamil. Click the radio button ○ next to the correct answer to mark it.
  • 5
    Fill in the Explanation — this is shown to the learner after they answer, whether right or wrong. Keep it concise and educational.
❓ True / False tip When you switch the type to True / False, the two options are auto-filled as True / Tamil and False / Tamil. You only need to set the correct answer and write the question.

Workflow Tab
Control who can see the chapter and track where it is in the review process.

Every chapter goes through a lifecycle. Use the Workflow tab to move it from one stage to the next.

draft → in_review → approved → published → archived
Status What it means Visible to learners?
draft Content is being written or edited No
in_review Sent for reviewer approval No
approved Reviewed and ready to go live No
scheduled Will go live at a set date/time No
published Live and accessible to all learners ✓ Yes
archived Retired — hidden from learners No
  • 1
    Open the Workflow tab. The current status is shown as a large badge.
  • 2
    Click one of the transition buttons to move the chapter to the next stage (e.g. → in_review).
  • 3
    A confirmation dialog will appear. Confirm to proceed.
  • 4
    When ready to make the chapter live, transition to published. It will immediately appear in the learner catalog.
⚠️ Can't skip stages You cannot jump from draft directly to published. The chapter must go through in_review → approved first. This ensures content is checked before going live.

Block Type Reference
Choose the right block for every type of content.
¶
Text
Paragraph of body copy
Use for regular explanatory paragraphs. Best for 2–4 sentences.
EN: Think of it like a bank locker — but for shares. When you buy shares of Tata Motors, they don't come as paper certificates.
TA: இதை ஒரு வங்கி லாக்கர் போல் நினைத்துக்கொள்ளுங்கள்…
H₁
Heading
Section title inside a card
Use to introduce a new sub-topic within a card. Keep it short — 3–5 words.
EN: What is a Demat account?   TA: Demat கணக்கு என்றால் என்ன?
💡
Tip
Helpful hint or pro advice
Use for practical advice or extra context that's good to know but not essential to the main flow.
EN: Popular broker apps: Zerodha, Groww, Angel One, Upstox. All SEBI registered.
TA: பிரபலமான broker ஆப்கள்: Zerodha, Groww, Angel One, Upstox.
📋
Callout / Note / Warning / Quote
Highlighted box for important content
Callout / Note — for key definitions or important facts.
Warning — for risks or things learners must not do.
Quote — for dialogue, examples, or memorable lines.
Warning EN: Never invest money you cannot afford to lose. Stock prices can fall.
TA: நீங்கள் இழக்க முடியாத பணத்தை முதலீடு செய்யாதீர்கள்.
📊
Fact Table
Key–value rows for structured facts
Use for step-by-step instructions, definitions, or any content that has a label and a value. Add rows with + Add Row.
Step 1 | படி 1 | Download a broker app | Broker ஆப் பதிவிறக்கம் செய்யுங்கள்
Step 2 | படி 2 | Enter mobile number and OTP | மொபைல் எண் கொடுத்து OTP சரிபாருங்கள்
⚖️
Comparison
Two or three columns side by side
Use to compare two concepts. Each column has an Emoji, Title (EN/TA), and Subtitle (EN/TA).
Col 1: ⚡ Market Order | Market ஆர்டர் | Buys instantly | உடனே வாங்கும்
Col 2: 🎯 Limit Order | Limit ஆர்டர் | Buys at your price | உங்கள் விலையில் வாங்கும்
🍕
Visual Comparison
Percentage/fraction visual rows
Use to show proportions visually (e.g. what fraction of a company 1 share represents). Set Total slices and add rows with percentage, emoji, label. Tick Highlight on the most important row.

Quick Tips
Things to remember while building content.
💾 Save often There is no auto-save. Click Save Metadata after editing details, and Save Lesson after editing cards. Your changes are lost if you navigate away without saving.
🌐 Always fill both EN and TA Every text field has an English and a Tamil side. Filling only English means Tamil learners will see a blank. Even a rough Tamil draft is better than empty.
❓ One quiz every 1–2 cards Learners engage better when quizzes break up long content. A 10-card lesson should have 3–4 quiz questions distributed throughout — not all at the end.
🔍 Check the live preview After publishing, visit the lesson URL (tradeze.in/learn/lesson.html?id=m1l4) to see exactly what learners see. The admin page only shows the raw fields.
⚠️ Don't publish incomplete chapters A chapter in draft is never visible to learners. Only move to published when the title, description, all cards, and at least one quiz question are complete.
🔄 You can always go back to draft Published chapters can be moved to archived, and archived chapters can return to draft. Nothing is permanent — you can always unpublish and fix content.
